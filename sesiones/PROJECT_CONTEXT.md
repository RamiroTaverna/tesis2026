# Contexto Maestro del Proyecto

## Proyecto activo

- Nombre de trabajo: `ByteCore`
- Tipo de solucion: sistema integral de gestion logistica y trazabilidad de paqueteria
- Dominio: PyMEs logisticas y de distribucion de ultima milla en Argentina
- Objetivo general: unificar en una sola plataforma la operacion de deposito, rutas, tracking, atencion al cliente y control operativo

## Fuentes base del workspace

- `Proyecto Final_ByteCore.pdf`: documento principal a terminar
- `PreProyecto_ByteCore.pdf`: documento aprobado, solo lectura, usado como antecedente y base conceptual
- `ideas/Proyecto Final Noah.pdf`: referencia de estructura y nivel de desarrollo esperado
- `ideas/PreProyecto Noah.pdf`: referencia secundaria
- `ideas/Pre y Proyectos/`: conjunto adicional de preproyectos y proyectos finales aprobados para comparar estructura, profundidad, continuidad y nivel de cierre

## Regla de continuidad documental

- `PreProyecto_ByteCore.pdf` no debe modificarse
- todo nuevo desarrollo debe concentrarse en la continuidad de `Proyecto Final_ByteCore.pdf`
- el preproyecto se usa para conservar coherencia academica, alcance, problema, enfoque y lenguaje del proyecto
- `ideas/` y `ideas/Pre y Proyectos/` sirven solo como referencia de forma, estructura y nivel de detalle de entregas aprobadas

## Sintesis del problema

El proyecto busca resolver ineficiencias comunes en empresas de logistica de ultima milla:

- falta de trazabilidad en tiempo real una vez que la unidad sale del deposito
- planificacion manual o empirica de rutas
- sobrecarga administrativa por consultas de tracking
- demoras y errores en prueba de entrega, rendiciones y cobros
- baja integracion entre areas operativas, comerciales y administrativas

## Alcance funcional declarado

El documento actual plantea un sistema tipo TMS, no un ERP contable. Incluye:

- recepcion y registro de paquetes
- clasificacion en deposito o nodo
- planificacion de rutas
- seguimiento en tiempo real
- gestion de entregas y devoluciones
- soporte documental operativo

Queda fuera de alcance:

- facturacion fiscal directa
- contabilidad general
- manufactura previa
- segmentos de carga regulada que requieran tratamientos especiales

## Huecos y debilidades detectadas en el proyecto final

- aparecen secciones solo nombradas pero no desarrolladas en profundidad
- faltan entregables tecnicos concretos para diseno y desarrollo
- varias partes parecen placeholders o notas internas
- no se ve un cierre editorial completo del documento

Pendientes que probablemente debamos construir o reforzar:

- aclaracion de la solicitud con mejor redaccion final
- informe de viabilidad preliminar
- anticipacion de requerimientos
- entrevistas, cuestionarios, observacion y analisis documental con formato academico
- plan estandar y plan de alto rendimiento
- estudio de factibilidad tecnica, economica y operativa
- proceso de decision con tabla y arbol
- descripcion grafica del sistema, diagrama 0 y 1
- diseno GUI
- entidad-relacion, modelo relacional y diccionario de datos
- diseno de entradas y salidas
- configuracion del entorno, codificacion, integracion y pruebas
- implementacion, migracion, capacitacion y mantenimiento
- indice final y revision de consistencia global

## Referencia util tomada de `ideas/`

El proyecto aprobado de referencia muestra algo importante: aunque no todo tiene un nivel tecnico extremo, si mantiene continuidad, desarrolla cada etapa con lenguaje academico y deja menos huecos entre secciones. El objetivo no es copiarlo, sino usarlo para:

- cerrar secciones incompletas
- sostener una estructura mas pareja
- evitar saltos bruscos entre una fase y otra
- convertir listas sueltas en contenido defendible

## Criterio de trabajo para futuras sesiones

- cada sesion debe producir un entregable concreto en Markdown
- antes de escribir, revisar si la seccion ya existe en el PDF y solo necesita expansion
- mantener consistencia entre preproyecto y proyecto final sin alterar el preproyecto aprobado
- si algo es supuesto o estimado, dejarlo explicitado
- priorizar primero cerrar el contenido faltante, luego mejorar forma y coherencia
