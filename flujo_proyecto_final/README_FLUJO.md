# Flujo del Proyecto Final

Esta carpeta concentra el flujo activo de trabajo del proyecto final para evitar mezclar contexto, checkpoints, estructuras y borradores.

## Convencion de prefijos

- `00-09`: contexto base y control general
- `10-19`: checkpoints y registro de sesiones
- `20-29`: estructura y guias maestras
- `30-39`: borradores de contenido redactado

## Archivos actuales

- `00_PROJECT_CONTEXT.md`: contexto maestro del proyecto
- `01_MASTER_CHECKLIST.md`: checklist general de avance
- `10_CHECKPOINT_ACTUAL.md`: estado de trabajo al cierre o apertura de sesion
- `11_SESSION_LOG.md`: historial resumido de sesiones
- `20_ESTRUCTURA_OBJETIVO.md`: mapa de estructura del proyecto final
- `21_GUIA_PASO_A_PASO_PROYECTO_FINAL.md`: guia integral de construccion
- `30_QUIENES_SOMOS_BYTECORE.md`: borrador institucional
- `31_PRESENTACION_CLIENTE_BYTECORE.md`: borrador de presentacion del cliente

## Regla de uso

Desde este punto, los nuevos documentos del proyecto final deben crearse preferentemente en esta carpeta, respetando los prefijos segun su funcion.

La carpeta `sesiones/` puede conservarse como respaldo historico, pero el flujo activo pasa a ser `flujo_proyecto_final/`.
